Error Boundaries & Fallbacks

This demo showcases how FuseWire handles errors in different component lifecycle hooks. Each child below is configured with a fallback: 'ErrorDemo/Fallback' option. When a child fails, the framework automatically renders the fallback and emits an fw-error event you can see in our Playground Console or your browser's one.

The Retry buttons do exactly the same thing, but this time we are re-adding the failed Component after the parent Component has been rendered (a more dynamic scenario).

1. Fails in init()

This component throws an error during the async init() hook, before it ever renders.

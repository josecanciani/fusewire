# Jison Parser Agent Guidelines

This directory contains the AST-based template expression parser for FuseWire.

## Important Constraints & Rules

1. **DO NOT EDIT `fusewire-expr.js` MANUALLY**
   - The `fusewire-expr.js` file is completely auto-generated by Jison. Any manual edits will be overwritten the next time the parser is built.
   - If you need to change the parsing logic, grammar, or operators, make your changes directly in `fusewire-expr.jison`.

2. **How to Build the Parser**
   - After modifying `fusewire-expr.jison`, you MUST run `npm run build:parser` from the project root.
   - This script generates the `fusewire-expr.js` file and appends necessary ES Module exports and linter/typescript ignore directives.

3. **Why `fusewire-expr.d.ts` is here**
   - You might wonder why a TypeScript definition file (`.d.ts`) is located in the `src/` directory instead of the `dist/` directory.
   - Because `src/template-compiler.js` imports `./parser/fusewire-expr.js` directly, the TypeScript compiler attempts to parse the enormous, auto-generated Jison JavaScript file to infer its types. This causes strict-mode typechecking errors and dramatically slows down the build.
   - By placing a handwritten `fusewire-expr.d.ts` right next to the JS file, TypeScript prioritizes reading this `.d.ts` file instead of parsing the JS file.
   - **DO NOT DELETE `fusewire-expr.d.ts`**. It is a critical workaround to keep the `npm run typecheck` script passing.

4. **Parser Architecture**
   - The parser relies on Jison to generate an AST (Abstract Syntax Tree) representing mathematical and logical operations within template bounds (e.g., inside `(( ... ))` or `fw-if="..."`).
   - The AST nodes returned must conform to the expected format in `src/template-compiler.js`, returning `type`, `operator`, `left`, `right`, `value`, etc.
